═══════════ play ═══════════
Play a Spotify URI or resume playback

When called without a URI, resumes the current playback.
When called with a Spotify URI, plays that specific content.
The URI must start with 'spotify:'.
Use --device to play on a specific device; playback is transferred
before starting. Prefer 'devices transfer' if you only want to move
playback without picking new content.

USAGE
  /Applications/Spotify.app/Contents/MacOS/spotify_cli play [<uri>] [flags]

FLAGS
  --device <name|id>      Play on a specific device (name or id from 'devices list').
                      Transfers playback before starting.

GLOBAL FLAGS
  --format text|json  Output format (default: text)
  -h, --help     Show help for command

EXAMPLES
  $ spotify_cli play
  $ spotify_cli play spotify:track:4iV5W9uYEdYUVa79Axb7Rh
  $ spotify_cli play spotify:track:4iV5W9uYEdYUVa79Axb7Rh --device "Kitchen Speaker"
  $ spotify_cli play --device "Kitchen Speaker"

═══════════ library batch ═══════════
Run a batch of mixed library, playlist, and folder operations

Run a sequence of mixed library operations in a single CLI invocation,
amortizing process-startup and connection-handshake cost that makes
single-op calls expensive for agent workflows.

Reads a JSON manifest from stdin (default) or --from-file. Each op is
dispatched sequentially against the same connection. Results are written
as one JSON line per op (always, regardless of --format), followed by a
final summary line in plain text or JSON (when --format json).

By default the batch continues on error; pass --stop-on-error to abort
at the first failure (remaining ops are reported with ok=false,
skipped=true). Exit code is non-zero if any op failed.

Manifest schema:
  {
    "stop_on_error": false,      // optional; --stop-on-error wins
    "ops": [
      {"op": "library_add",     "uris": ["spotify:..."]},
      {"op": "library_remove",  "uris": ["spotify:..."]},
      {"op": "library_contains","uris": ["spotify:..."]},
      {"op": "playlist_create", "name": "...",
                                    "description": "...", "public": false,
                                    "image_file": "/path/to/cover.jpg"},
      {"op": "playlist_update", "playlist_uri": "spotify:...",
                                    "name": "...", "description": "...",
                                    "public": true,
                                    "image_file": "/path/to/cover.jpg"},
      {"op": "playlist_add",    "playlist_uri": "spotify:...",
                                    "track_uris": ["spotify:..."], "position": 0},
      {"op": "playlist_remove", "playlist_uri": "spotify:...",
                                    "positions": [0,1,5]},
      {"op": "folder_create",   "name": "...",
                                    "parent_folder_uri": "spotify:..."},
      {"op": "folder_rename",   "folder_uri": "spotify:...", "name": "..."},
      {"op": "folder_move",     "uris": ["spotify:..."],
                                    "destination_folder_uri": "root"},
      {"op": "folder_remove",   "folder_uri": "spotify:...",
                                    "keep_contents": false}
    ]
  }


USAGE
  /Applications/Spotify.app/Contents/MacOS/spotify_cli library batch [--from-file <path>] [--stop-on-error] [--format json]

FLAGS
  --from-file <path>      Read JSON manifest from file (default: stdin)
  --stop-on-error         Abort the batch on the first failure

GLOBAL FLAGS
  --format text|json  Output format (default: text)
  -h, --help     Show help for command

EXAMPLES
  $ spotify_cli library batch --from-file ops.json
  $ echo '{"ops":[{"op":"library_add","uris":["spotify:track:abc"]}]}' | spotify_cli library batch
  $ spotify_cli library batch --from-file ops.json --stop-on-error --format json
